## 📁 This repository

Problem notes (problem → pattern → brute → optimize → template → code → edges → variants → reuse).

| Step | In one sentence — what you decide |
|------|-------------------------------------|
| **P** — Problem | Inputs, exact output, and guarantees (e.g. sorted, exists, unique). |
| **P** — Pattern | What repeats? Pair / frequency / window / prefix? **Name the pattern.** |
| **B** — Brute | Stupid way in one line → **time** and **space**. |
| **O** — Optimize | What are you re-asking? **Cache, skip, or invert** the question. |
| **T** — Template | 2–4 bullets: state you keep, loop order, update rule, stop/return. |
| **C** — Code | Write from **T** only; match naming of the repo / language. |
| **E** — Edge | `n=0/1`, dupes, negatives, overflow, “no answer” if allowed. |
| **V** — Variants | Sorted array? 1-based? return all? k-generalization? |
| **R** — Real World Applications | Where can such a pattern be used? What use-cases? |

**Practice deliberately — pattern first, then code.**
